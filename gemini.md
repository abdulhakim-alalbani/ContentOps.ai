# Agency Client Dashboard - Project Overview

## 📌 What is this app?
This application is a **Client-facing Dashboard** designed for an agency. It allows the agency's clients to log in and monitor the progress of their ongoing tasks, view analytics/spend, review delivered content, request revisions, and communicate via an inbox. 

It acts as a premium, transparent portal for clients to see what the agency is currently working on. **Clients do not have the ability to start work or create new orders from scratch themselves**; they only observe, track, and provide feedback on tasks generated by the agency.

## 🛠️ Technology Stack
- **Framework:** Next.js (App Router)
- **Styling:** Tailwind CSS
- **Icons:** `lucide-react`
- **Internationalization (i18n):** `next-intl`
- **Fonts:** `Outfit` (Primary Sans-serif)

## ✅ What has been accomplished so far?
We have transformed a basic template into a modern, premium web application with interactive components. 

1. **Design System & Aesthetics Overhaul:**
   - Transitioned the entire app to the `Outfit` font for a modern look.
   - Refactored the UI from using heavy drop shadows to a **flat, border-driven aesthetic** (e.g., `border border-border` without `shadow-sm`) to achieve a premium, cutting-edge feel.
   - Standardized text casing: All major titles and UI elements were converted to **Sentence case** instead of Title Case.

2. **Core Pages Implemented:**
   - **Dashboard Overview (`/dashboard`):** Features key metrics (Total spend, tasks in progress), a dynamic task completion chart, and a table for "Recent tasks" complete with avatars, status tags, and deadlines. **All table rows are now fully clickable and route to task details.**
   - **My Tasks (Kanban Board) (`/orders`):** A robust Kanban board displaying tasks in various stages (To do, In progress, In review, Completed). Features a working search filter to find tasks by title, description, or status. **All task cards are fully clickable.**
   - **Task Details (`/orders/[id]`):** A dynamic view showing the brief, timeline status (Submitted -> In Progress -> Review -> Completed), delivered content, and actions for the client to "Approve" or "Request Revision" with modals.
   - **Inbox (`/inbox`):** A messaging interface for clients to communicate with the agency. Includes "Mark all as read" functionality.
   - **Settings (`/settings`):** Fully designed layout for Profile and App-specific settings.

3. **UI/UX Refinements:**
   - **Cleaned Navigation:** Removed unused and irrelevant sidebar links (Calendar, Reports, Portfolio, Team management, etc.) to streamline the user experience to exactly what the client needs.
   - **Topbar Interactivity:** Added a clickable Notification Bell with a dropdown showing unread messages. Made the User Profile icon a clickable dropdown menu with links to Settings and Sign out.
   - **Dynamic Mockups:** Configured the mock-data layer to auto-generate placeholder task detail pages for any arbitrary task ID clicked from the Kanban board.

## 🤖 Instructions for Future AI Models

If you are a new AI model or agent picking up this project, please strictly adhere to the following guidelines:

### 1. Design Language (CRITICAL)
- **No Drop Shadows:** The design language explicitly avoids `shadow-md` or `shadow-sm`. Rely exclusively on precise borders (e.g., `border border-border` or `border-gray-200`) and rounded corners (`rounded-xl` or `rounded-2xl`).
- **Typography:** Ensure you are using the `Outfit` font. Keep all new titles, buttons, and headers in **Sentence case** (e.g., "Recent tasks" not "Recent Tasks").
- **Colors:** Primary brand color is a vibrant orange/red (`#f93a20`). Use Tailwind gray scales (`gray-50` to `gray-900`) for depth and contrast.

### 2. File Structure & Routing
- The project relies on Next.js localized routing (`[locale]`). Ensure all new pages are created within `src/app/[locale]/(dashboard)/...`.
- Keep the `Sidebar.jsx` and `Topbar.jsx` strictly limited to pages that actually exist. Do not add arbitrary links without confirming page creation.

### 3. Internationalization
- All static text should be routed through `next-intl`. 
- When adding new text to the UI, add the string to `messages/en.json` first, and use the `useTranslations` hook in the component.

### 4. Data Handling
- The app currently uses a mock data layer located at `src/lib/mock-data.js`. 
- If you need to add or mutate tasks, users, or settings, use the helper functions in this file (e.g., `getOrderById`, `updateOrderStatus`). DO NOT hardcode new arrays inside components if they need to be shared across pages.
- The `getOrderById` function is designed to return a generic mockup if a specific ID isn't found. Rely on this fallback to prevent 404s during development.

### 5. Icons
- Strictly use `lucide-react` for all icons. Do not import random SVG strings unless absolutely necessary.

By following these instructions, you will ensure the dashboard remains consistent, premium, and fully aligned with the established client expectations.

# نظرة عامة على المشروع (النسخة العربية)

## 📌 ما هو هذا التطبيق؟
هذا التطبيق عبارة عن **لوحة تحكم خاصة بالعملاء** مصممة لصالح وكالة. تتيح اللوحة لعملاء الوكالة تسجيل الدخول ومتابعة تقدم مهامهم الحالية، عرض الإحصائيات والمصروفات، مراجعة المحتوى المُسلّم، طلب تعديلات، والتواصل عبر صندوق الوارد.

يعمل التطبيق كبوابة احترافية وشفافة للعملاء لرؤية ما تعمل عليه الوكالة حالياً. **لا يمتلك العملاء صلاحية بدء عمل جديد أو إنشاء طلبات من الصفر بأنفسهم**؛ دورهم يقتصر على المراقبة، التتبع، وتقديم الملاحظات على المهام التي تنشئها الوكالة.

## 🛠️ التقنيات المستخدمة
- **إطار العمل:** Next.js (App Router)
- **التصميم:** Tailwind CSS
- **الأيقونات:** `lucide-react`
- **تعدد اللغات:** `next-intl`
- **الخطوط:** `Outfit`

## ✅ ما تم إنجازه حتى الآن؟
لقد قمنا بتحويل قالب أساسي إلى تطبيق ويب حديث واحترافي بمكونات تفاعلية.

1. **تحديث النظام البصري (Design System):**
   - الانتقال بالكامل لاستخدام خط `Outfit` لمظهر عصري.
   - إعادة بناء الواجهات للاعتماد على **الحواف (Borders)** بدلاً من الظلال (Shadows) الكثيفة، لتحقيق طابع احترافي ومسطح (Flat Design).
   - توحيد حالة الأحرف لتصبح (Sentence case) في جميع العناوين والمكونات بدلاً من (Title Case).

2. **الصفحات الأساسية:**
   - **الرئيسية (`/dashboard`):** تعرض المقاييس الرئيسية، رسم بياني ديناميكي لتقدم المهام، وجدول للمهام الحديثة (Recent tasks). **جميع صفوف الجدول قابلة للنقر الآن وتأخذك لتفاصيل المهمة.**
   - **مهامي (`/orders`):** لوحة مهام (Kanban Board) تفاعلية. تحتوي على فلتر بحث يعمل بالكامل للعثور على المهام. **جميع المهام في اللوحة قابلة للنقر.**
   - **تفاصيل المهمة (`/orders/[id]`):** عرض ديناميكي لملخص المهمة، حالة التقدم، المحتوى المُسلم، وأزرار لاتخاذ إجراءات مثل الموافقة (Approve) أو طلب تعديل (Request Revision) مع نوافذ منبثقة (Modals).
   - **صندوق الوارد (`/inbox`):** واجهة مراسلات للتواصل بين العميل والوكالة.
   - **الإعدادات (`/settings`):** صفحة بتصميم كامل لإعدادات الحساب والتطبيق.

3. **تحسينات واجهة المستخدم (UI/UX):**
   - **تنظيف القوائم:** إزالة الروابط غير المستخدمة من القائمة الجانبية (Sidebar) للتركيز فقط على ما يهم العميل.
   - **تفاعل الشريط العلوي (Topbar):** إضافة جرس إشعارات تفاعلي يعرض قائمة بالرسائل غير المقروءة، وقائمة منسدلة للحساب الشخصي.
   - **بيانات افتراضية ذكية (Dynamic Mockups):** برمجة قاعدة البيانات المؤقتة لإنشاء صفحة تفاصيل افتراضية لأي مهمة يتم النقر عليها لتجنب ظهور صفحة الخطأ 404.

## 📱 هل التطبيق متجاوب (Responsive) بالكامل؟
التطبيق يستخدم نظام الـ Grid في Tailwind CSS لترتيب المحتوى الداخلي بشكل متجاوب (يتغير ترتيب الكروت والإحصائيات بناءً على حجم الشاشة). **ولكن، التطبيق مُحسّن حالياً بشكل رئيسي لشاشات الكمبيوتر (Desktop) والأجهزة اللوحية (Tablets).**
بالنسبة لشاشات الهواتف المحمولة الصغيرة، القائمة الجانبية (Sidebar) حالياً ثابتة العرض ولا تحتوي على زر الإخفاء/الإظهار (Hamburger menu)، مما قد يؤثر على تجربة الاستخدام في الهواتف. يتم التخطيط لإضافة هذه الميزة لاحقاً لضمان تجربة مثالية على جميع الأجهزة.
